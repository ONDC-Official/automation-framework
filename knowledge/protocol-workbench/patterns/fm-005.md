# D-005 — UI flow-trigger broken (submodule skew)

## Signature
- Starting a flow from the UI does nothing / ui-backend returns 500; mock log shows 404 on `/mock/{d}/{v}/trigger/api-service/<action>`.

## Quick checks
1. mock log for `404` on a `trigger/api-service/...` path ⇒ confirmed skew.
2. compare ui-backend vs mock submodule commits against `references/kb-commit-baseline-*.md`.

## Cause
- ui-backend (automation-frontend) is on a version that calls the OLD mock route `trigger/api-service/:action`; the rewritten mock no longer serves it ⇒ 404 ⇒ ui-backend 500. The live mock initiates via `/flows/new` + `/manual/:action` instead. NOTE: origin/main still carries this stale route (re-pin alone doesn't fix it).

## Fix locus
- align the ui-backend↔mock submodule commits (or patch the ui-backend trigger route to the current mock API). For now, drive flows via the mock's `/flows/new` + `/manual` (the runtime path), not the UI trigger.

## Verify
- UI "trigger" reaches the mock without 404; or `POST /mock/playground/flows/new` succeeds directly.
