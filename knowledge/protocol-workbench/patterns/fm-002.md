# D-002 — flow dies after ~5 min / empty HTML form

## Signature
- UI flow "stops working" a few minutes in; `/flows/current-status` and `/forms/...` return 500 "No transaction data found for transaction ID … and subscriber URL …"; an HTML_FORM modal renders empty (only Submit).

## Quick checks (ordered)
1. **How long since the last step?** If ~5 min → recorder TTL. `redis-cli -n 0 TTL "<txn>::<suburl>"` (the apiList key). Gone/short ⇒ this is it.
2. **Was on_search NACK'd?** (empty form, not TTL) check api-service log for on_search NACK on `REGEX_CONTEXT_BAP_ID` — local `bap_id` = host.docker.internal:3032 (with port) trips L1. No catalog/form-url saved ⇒ empty form.

## Cause
- (1) `transaction_id::subscriber_url` apiList TTL = `RECORDER_CACHE_TTL_SECONDS_DEFAULT`, pinned to **300s** in docker-env ⇒ cache vanishes ~5 min after last write (while `FLOW_STATUS_` ~5h and `MOCK_DATA::` -1 survive). The form endpoint reads the now-missing txn cache.
- (2) protocolValidations=true + local bap_id port ⇒ on_search NACK ⇒ BPP catalog/form-url never saved ⇒ form empty.

## Fix locus
- (1) `docker-env/recorder-service.env`: `RECORDER_CACHE_TTL_SECONDS_DEFAULT` (+`RECORDER_API_TTL_SECONDS_DEFAULT`) 300 → **18000**; `docker compose up -d recorder-service` (env-only).
- (2) set session difficulty `protocolValidations=false` (UI "Protocol Validation" toggle). Do NOT "fix" the local bap_id — host:port is required for local.

## Verify
- fresh flow: txn cache persists past 5 min; form modal populates (gold-loan form ~12KB with fields).
