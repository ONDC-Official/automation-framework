# D-006 — local cloud-frontend (release-eks-tech) gotchas

## Signature
- The cloud/EKS-targeted frontend run locally on `http://host.docker.internal:3035` fails with: CORS preflight blocking `/api/sessions`; `crypto.randomUUID is not a function`; or cross-origin to `localhost:3034`.

## Causes + fixes (each independent)
1. **CORS**: ui-backend `allowedHeaders` (app.ts) lacked `x-api-key` though the frontend sends it ⇒ preflight blocks. Fix: add `x-api-key` to allowedHeaders, gated `NODE_ENV==="development"` (off in prod). 
2. **crypto.randomUUID undefined**: `http://host.docker.internal` is NOT a secure context ⇒ `crypto.randomUUID`/`subtle` unavailable. Fix (instant): open via `http://localhost:3035` (localhost = secure). Fix (proper): replace with `uuidv4()` (done) or polyfill.
3. **cross-origin**: page at host.docker.internal:3035 but `VITE_BACKEND_URL=localhost:3034`. Use one host consistently (localhost).

## Meta-pattern
- These are LOCAL-only artifacts of a cloud-targeted build (HTTPS/real-domain assumptions). In real EKS (https, real domain) none of them occur. Don't "fix" the cloud build for them beyond dev-gated shims.

## Verify
- open the UI at `http://localhost:3035`; sessions load; form submit no longer throws.
